# Настройка стилей для компонента Media

## Проблема с отображением

Если компонент Media отображается некорректно (неправильные цвета, отсутствие стилей), это означает, что CSS переменные не подключены.

## Решение

### Шаг 1: Публикация стилей

```bash
php artisan vendor:publish --tag=media-styles
```

Это создаст файл `resources/css/vendor/media.css` с необходимыми CSS переменными.

### Шаг 2: Подключение стилей

#### Вариант 1: В главном CSS файле (рекомендуется)

Добавьте в `resources/css/app.css`:

```css
@import '../css/vendor/media.css';
```

#### Вариант 2: В главном JS файле

Добавьте в `resources/js/app.js`:

```javascript
import '../css/vendor/media.css'
```

#### Вариант 3: В компоненте напрямую

Добавьте в компонент, где используется Media:

```vue
<script setup>
import Media from '@/vendor/media/components/Media.vue'
import '@/css/vendor/media.css'
</script>
```

### Шаг 3: Пересборка фронтенда

Если используете Vite:

```bash
npm run build
# или для разработки
npm run dev
```

## Кастомизация цветов

Если хотите изменить цветовую схему, переопределите CSS переменные в своем `app.css`:

```css
:root {
    --accent: 210 40% 96.1%; /* Ваш цвет акцента */
    --background: 0 0% 100%; /* Ваш цвет фона */
    --foreground: 222.2 84% 4.9%; /* Ваш цвет текста */
    --card: 0 0% 100%; /* Цвет карточек */
    --border: 214.3 31.8% 91.4%; /* Цвет границ */
    --destructive: 0 84.2% 60.2%; /* Цвет для удаления */
    /* ... другие переменные */
}
```

## Проверка

После подключения стилей компонент должен отображаться с правильными цветами:
- Фон должен быть белым (или темным в dark mode)
- Кнопки должны иметь правильные цвета
- Текст должен быть читаемым

## Дополнительно

Убедитесь, что в вашем проекте установлен и настроен Tailwind CSS, так как компонент использует Tailwind классы для layout и spacing.
